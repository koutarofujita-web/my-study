# インフラ・セキュリティ＆プログラミング学習記録

本日は、paizaでのアルゴリズム問題（標準入力処理）の実践から、Linux（WSL/Ubuntu）環境における「ハッシュ関数を用いたセキュリティ解析」および「コマンドによるファイルシステムの完全支配」まで、幅広いレイヤーの技術を網羅して学習しました。

## 🎯 本日の学習成果・習得コマンド

### 1. プログラミング基礎（Python3）
- **標準入力（`input()`）のマスター**：外部から動的に与えられる文字列データをスクリプト内で正確にキャッチし、制御する実装力を習得。
- **データ型変換とデバッグ**：文字列型（str）と整数型（int）の厳格な違いを理解し、型エラー（TypeError/NameError）の原因を的確に突き止めて修正するデバッグスキルを強化。

### 2. インフラ・セキュリティ（Linux/Ubuntu）
| コマンド | 用途・本日実践した操作 |
| :--- | :--- |
| `sha256sum` | 文字列を世界基準の暗号アルゴリズム（SHA-256）でハッシュ化する |
| `mkdir` | ターミナル上で新しいディレクトリ（フォルダ）を一瞬で爆誕させる |
| `cd` | ディレクトリ間をマウスを使わずに一瞬でワープ（移動）する |
| `touch` | 中身が空っぽの新規ファイルを右クリックなしで高速生成する |
| `echo "text" > file` | リダイレクト（`>`）を使い、ファイル内へ文字を直接流し込む |
| `cat` | ファイルの中身をエディタで開くことなくターミナル上に覗き見る |


## 🔬 実践セッションの考察と気づき

### ① 暗号の「雪崩効果（Avalanche Effect）」の検証
- 自分の名前（`koutarou`）を `sha256sum` でハッシュ化し、16進数（0〜9, a〜f）で構成された64文字の強固な暗号の生成を確認。
- 先頭の1文字を大文字（`Koutarou`）に変えて再度実行したところ、出力が `94fbe...` から `e4c4d9...` へと跡形もなく激変する現象（雪崩効果）を自らの手で実証。
- この特性が、Webサイトにおける「パスワードの安全な保管（不可逆暗号）」や、ウイルス対策ソフトにおける「ファイルの指紋（シグネチャ）検知による不正改ざんの見破り」にどう直結しているのか、セキュリティエンジニアの必須知識を腹落ちさせました。

### ② コマンドラインによるファイルシステムの支配
- ターミナルのみで `secret_base` ディレクトリを作成し、その内部へ移動（`cd`）。さらに `mission.txt` を生成してテキストを書き込み、`cat` で中身を確認するという一連のLinuxオペレーションコンボを完全ノーミスで達成。
- プロンプトの現在地表示（`~/secret_base$`）の変化を捉え、GUI（マウス操作）に頼らない、CUIによる高速なサーバー構築・運用の基礎体力を身につけました。
