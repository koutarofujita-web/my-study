## 1. 環境構築の壁とトラブルシューティング
* **事象:** ChromeOS Flex上のLinux（Debian）環境でDockerを起動しようとした際、`A dependency job for docker.service failed.` というエラーが発生。
* **原因:** OSの深いレイヤー（システムサービス）の依存関係による一時的な不整合。
* **解決策:** Linux環境およびPC本体を一度再起動することで、Dockerサービスが正常に連動して起動（`Active: active (running)`）することを確認した。

## 2. Dockerコンテナ内でのセキュリティツール実験
* **やったこと:** まっさらなUbuntuコンテナ（使い捨ての実験室）を起動し、ハッカー対策・ネットワーク診断ツールである `nmap` をインストール。`nmap localhost` コマンドでネットワークポートの診断を実行した。
* **学び:** `exit` でコンテナを抜けると、インストールしたツールや設定が全て自動でリセットされる（不変・使い捨ての特性）を体感、利便性を学んだ。

## 3. Webサーバー（Nginx）の一瞬構築
* **やったこと:** `sudo docker run -d -p 8080:80 --name my-webserver nginx` コマンドを実行。裏方（バックグラウンド）で世界標準のWebサーバーを1秒で起動させた。
* **成果:** Chromeブラウザから `http://localhost:8080` にアクセスし、「Welcome to nginx!」の画面が表示されることを確認。実務におけるコンテナを使ったWebサイト公開の流れを体感した。
* **管理:** 実習後は `docker stop` および `docker rm` を使い、サーバーを安全に停止・お片付けする運用の基礎を実践した。

## 4. コンテナ内でのC言語開発環境の構築
* **やったこと:** 再びUbuntuコンテナを立ち上げ、プログラミングに必要な開発ツール一式（`build-essential`）とテキストエディタ `vim` をインストール。
* **壁の突破:** インストール時のタイムゾーン質問（Geographic area）に対し、アジア（`5`）の東京（`77`）を的確に選択して突破。また、Vimの入力モード切り替えエラー（`E348: No string under cursor`）を `Esc` 連打と小文字の `i`（挿入モード）で冷静に解決した。
* **成果:** C言語でソースコード（`hello.c`）を執筆し、`gcc` でコンパイル（翻訳）を実行。`./hello` コマンドにて、画面に `Docker de C-Gengo Seiko!!` と出力させることに一発で成功した。
