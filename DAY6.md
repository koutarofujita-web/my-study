# DAY6: DockerによるWebサーバー（nginx）構築とコンテナ運用の効率化

Linux環境（WSL）上に、軽量な仮想化技術である**Docker**を使ってWebサーバー（nginx）を立ち上げました。さらに、実務で必須となる「コンテナの外からデータを流し込む（ボリュームマウント）」という効率的な運用方法に挑戦しました。

---

## 🎯 今回のゴール
1. Dockerコンテナでnginxを起動し、ブラウザからアクセスできるか検証する。
2. コンテナ内の直接編集から、ホスト側のディレクトリと同期させる効率的な方法（ボリュームマウント）へシフトする。

---

## 💻 実践した内容とコマンド

### 1. nginxコンテナの起動と生存確認
まずは、過去の学習で作成したnginxコンテナが動いているか状態を確認しました。停止していたため、コンテナを起動させて生存確認（アクセス確認）を行います。

```bash
# 1. コンテナの状態（停止中も含めて）を確認する
docker ps -a

# 2. 停止していたサーバーコンテナを起動する
docker start my-server

# 3. 再度コマンドを打ち、正しく起動（Up）しているか確認する
docker ps
```

ブラウザで `http://localhost` にアクセスし、nginxの初期画面が表示されることを確認しました。
